GitHub 使い方まとめ（基礎・実用）

◆ 基本の操作フロー（毎回これだけ）

1.  変更を登録する（add）
        git add .

2.  状態を記録する（commit）
        git commit -m "メッセージ"

3.  GitHub にアップロード（push）
        git push origin main

------------------------------------------------------------------------

◆ よく使う Git コマンド一覧

◎ 状態確認
    git status

◎ GitHub から最新を取得（pull）
    git pull origin main

◎ 過去の履歴を見る
    git log

◎ ステージ取り消し
    git reset

◎ ファイル削除
    git rm ファイル名
    git commit -m "remove file"
    git push

------------------------------------------------------------------------

◆ 新しくリポジトリを作る流れ
    git init
    git add .
    git commit -m "first commit"
    git branch -M main
    git remote add origin https://github.com/ユーザー名/リポジトリ名.git
    git push -u origin main

------------------------------------------------------------------------

◆ 絶対に push してはいけないファイル
-   token.json
-   client_secret.json
-   .env
-   API Key を含むファイル

→ .gitignore に追加してブロックする

------------------------------------------------------------------------

◆ VSCode での Git 操作
-   左の「Source Control」アイコン
-   変更ファイルの一覧
-   commit メッセージ入力 → Commit
-   Sync Changes → push/pull 同時実行

------------------------------------------------------------------------

◆ よくあるトラブル対処

◎ リモートに履歴があり push できない
    git push -f origin main
※ 基本はやらない、初期化した時だけOK

◎ ファイルを無視したい（.gitignore）
    venv/
    __pycache__/
    token.json
    client_secret.json
    .env

------------------------------------------------------------------------
◆ GitHub を使うメリット
-   バックアップが取れる
-   バージョン管理ができる
-   共同開発しやすい
-   ポートフォリオになる
